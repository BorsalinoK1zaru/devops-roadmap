## День 2. Теория

PowerShell — оболочка Windows. В ней я могу запускать wsl, docker, git и Windows-команды.

WSL — Linux-среда внутри Windows. В ней я выполняю Linux-команды: uname, ls, cat, grep, chmod, apt.

Docker container — изолированная среда для запуска приложения. Контейнер создаётся из образа.

Docker image — шаблон, из которого создаются контейнеры.

docker run создаёт и запускает контейнер.

Проброс портов 8080:80 означает: порт 8080 на моём компьютере перенаправляется на порт 80 внутри контейнера.

docker ps показывает запущенные контейнеры.

docker logs показывает логи контейнера.

docker exec позволяет выполнить команду внутри контейнера.

docker stop останавливает контейнер.

docker rm удаляет контейнер.

## День 2. Что я понял

1. Чем PowerShell отличается от WSL? - PowerShell - обололчка ОС Windows, где я пишу комманды, WSL - Linux
2. Где выполняется команда `uname`? - WSL
3. Что делает команда `docker run -d -p 8080:80 nginx`? - создаю и запускаю контейнер в фоне на порте 8080 (на моей машине), и прослушиваю порт 80 внутри контейнера  
4. Что означает `8080:80`? 8080 - порт моей машины, 80 - порт контейнера, который я собираюсь прослушивать
5. Чем контейнер отличается от виртуальной машины? Виртуальная машина запускает полноценную отдельную ОС со своим ядром. Контейнер не запускает отдельную ОС, а изолирует приложение и его окружение, используя ядро хостовой системы. Контейнер легче и быстрее VM.
6. Как зайти внутрь контейнера? - docker exec -it devops-nginx sh
7. Где лежит стандартная HTML-страница Nginx внутри контейнера? /usr/share/nginx/html/index.html

## День 3. Что я понял

- Docker image — это шаблон для создания контейнера.
- Docker container — это запущенный экземпляр image.
- Volume mount позволяет подключить папку с моей машины внутрь контейнера.
- В команде `-v ${PWD}:/usr/share/nginx/html` моя текущая папка заменяет стандартную папку сайта Nginx внутри контейнера.

## День 4. Что я понял

- Dockerfile — это инструкция для сборки Docker image.
- `FROM nginx:latest` берёт готовый Nginx image как основу.
- `COPY index.html /usr/share/nginx/html/index.html` копирует мой HTML-файл внутрь image.
- `docker build` собирает image.
- `docker run` запускает container из image.
- Tag нужен, чтобы различать версии image, например `my-nginx-site:day4`.
- Точка в `docker build -t my-nginx-site:day4 .` означает текущую папку как build context.

## День 5. Что я понял

- Docker image состоит из слоёв.
- Инструкции `FROM`, `LABEL`, `RUN`, `COPY` участвуют в сборке image.
- Docker cache позволяет не пересобирать неизменённые шаги.
- Если файлы не менялись, повторный `docker build` проходит быстрее.
- Если изменить `index.html`, слой с `COPY index.html ...` должен быть пересобран.
- `docker history` показывает историю слоёв image.
- Порядок инструкций в Dockerfile важен: редко меняющиеся шаги лучше ставить выше, часто меняющиеся — ниже.
