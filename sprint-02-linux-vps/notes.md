## День 21. Что я понял

- SSH нужен для подключения к удалённому Linux-серверу.
- Private key хранится локально и никому не передаётся.
- Public key можно добавлять на сервер или в панель VPS.
- chmod 600 нужен для приватного ключа.
- chmod 700 нужен для папки ~/.ssh.
- whoami, id и groups помогают понять текущего Linux-пользователя.

## День 22. Что я понял

- `ls -l` показывает права, владельца, группу и размер файла.
- `chmod` меняет права доступа.
- `chmod +x` делает скрипт исполняемым.
- `ps` показывает процессы.
- PID — идентификатор процесса.
- `kill PID` останавливает процесс.
- `ss -tulpn` показывает listening ports и процессы.
- `curl` помогает проверить HTTP-сервис.
- `tail -f` позволяет смотреть логи в реальном времени.
  

  ## День 23. Что я понял

- Bash-скрипт — это файл с командами терминала.
- Shebang `#!/usr/bin/env bash` указывает, чем запускать скрипт.
- `chmod +x` делает скрипт исполняемым.
- Exit code `0` означает успех, не `0` означает ошибку.
- `$1` — первый аргумент скрипта.
- `${1:-default}` задаёт значение по умолчанию.
- `set -euo pipefail` помогает писать более строгие скрипты.
- `curl -f` полезен для healthcheck.
- `ss -tulpn` помогает проверить, слушает ли сервис порт.
- `tail -n` показывает последние строки лога.
